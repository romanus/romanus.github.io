The hypothesis for the $t$-test:

$\begin{cases} H_0: \beta_{LotArea} = 0, \text{ (}\texttt{LotArea}\text{ has no impact on the model)} \\ H_1: \beta_{LotArea} \ne 0 \end{cases}$

The corresponding $t$-static:

$t_{emp} = \frac{\widehat{\beta_{LotArea}} - \beta_{LotArea}}{\sqrt{\widehat{Var(\beta_{LotArea})}}} \sim t_{N-K-1}$,

where N is the sample size, K is the number of model coefficients.

We are going to pick the $t$-statistic and the $p$-value from the software:

Since our parameter follows the $t$-distribution, the 95% CI ($\alpha=0.05$) can be computed as:

$H_0: \beta_{LotArea}^* = 0$

$CI_{1-\alpha/2} = \beta_{LotArea}^* + t_{N-K-1, 1-\alpha/2} \sqrt{\widehat{Var(\beta_{LotArea})}} $

$CI_{\alpha/2} = \beta_{LotArea}^* - t_{N-K-1, 1-\alpha/2} \sqrt{\widehat{Var(\beta_{LotArea})}} $

# adapted from https://datascience.stackexchange.com/a/24447
def stepwise_selection(X, y, initial_list=[], alpha = 0.05, verbose=True):
    included = list(initial_list)
    while True:
        changed=False

        model = sm.OLS(y, sm.add_constant(pd.DataFrame(X[included]))).fit()
        
        # backward step
        pvalues = pd.Series(dtype=np.float64)
        rsquareds = pd.Series(dtype=np.float64)

        # use all coefs except intercept
        for col in model.params.index[1:]:
          included_reduced = filter(lambda key: key != col, included)
          model_reduced = sm.OLS(y, sm.add_constant(pd.DataFrame(X[included_reduced]))).fit()
          model_reduced_df = model_reduced.df_model
          fvalue = (model_reduced.df_resid - model_reduced_df) * (model.rsquared - model_reduced.rsquared) / (1 - model.rsquared)
          rsquareds[col] = model_reduced.rsquared
          pvalues[col] = 1 - st.f.cdf(fvalue, 1, model_reduced_df)

        worst_feature = pvalues.index[pvalues.argmax()]
        if pvalues[worst_feature] > alpha:
            changed=True
            included.remove(worst_feature)
            if verbose:
                print('Drop {:20} with p-value {:>12.6}; the updated R^2: {:>.6}'.format(worst_feature, pvalues[worst_feature], rsquareds[worst_feature]))
        if not changed:
            break
    return included

Explanation

Here we test all nested models using the $F$-test statistics:

$\begin{cases} H_0: R_{M_{out}}^2 = R_M^2 \text{ (both models are equivalent) } \\ H_1: R_{M_{out}}^2 < R_M^2 \text{ (model }M_{out}\text{ is worse) }\end{cases}$

$F-to-remove = \frac{R_M^2 - R_{M_{out}}^2}{1 - R_M^2} \sim F_{1;N-K_1}$

So, for each feature of the dataset we test if removing this feature from the model makes the model statistically different. Then we take the feature with worst $p$-value, for which we still cannot reject $H_0$ and drop it.

At the last iteration removing any of the features would reject $H_0$ and our model would become statistically different $\implies$ the model is selected.

We consider now the model with $\texttt{SalePrice}$ and the model with $\texttt{log(SalePrice)}$. Run an appropriate test to decide which of the models is superior. Explain, the idea of the test and why you cannot make a similar decision using AIC/BIC, etc.

We need to use the $J$-test here to compare two models:

$y = SalePrice = \beta_0 + \sum \beta_i X_i$

$y^{log} = log(SalePrice) = \beta_0^{log} + \sum \beta_i^{log} X_i$

Then let's consider the next connection between the models:

$e^{\hat{y_{log}}} = z_i'\gamma + \delta \hat{y} + v_i$

The hypothesis:

$H_0: \delta=0$ (the log model is not supperior)

Task 1d¶

What kind of problems might arise if we decide to fit a non-linear regression using all explanatory variables?

  1. All our statistical methods (tests, correlation measures etc.) are not valid anymore.
  2. Modeling: we don't know the true nonlinearity nature, so cannot create a correct model (even in a simple case of only $\texttt{YearBuilt}$ we are not sure which model to pick, what to do with 38 variables?).
  3. Modeling: we need to provide a valid first guess, and problems are the same when creating a model.
  4. Mathematical: when using iterative methods, we need to tune them not to stop at some local extremums (but high learning rates are bad, too).
  5. Overflows: we might get infinities when computing matrix pseudoinverses in close-to-singular cases.
  6. Timing: it might take some time for the model to converge (especially having small learning rates).

Task 2a¶

An important calibration parameter of a nonparametric regression is the bandwidth. Explain what happens with the regression/the weights in the Nadaraya-Watson regression if the bandwidth is too high or too small.

Bandwidth too small: undersmoothing

The bandwidth captures very few points at one time, the density plot will look like a combination of individual peeks (one peek per each sample element).

Bandwidth too high: oversmoothing

The model changes too slowly to capture all changes, the density plot is far from the true histogram. This means the variance would be huge and we cannot capture anything specific about this distribution (it looks more or less homogenous).
